Page Basics

Anything using the ADF will need to have a ancestor element with the class .adf-page for the code to initialize against. Everything utilizing ADF will need to be inside that element.

Regions

Each piece of the page within that .adf-page parent will need to be wrapped in an element with the .adf-region class on it. The app will iterate through those regions and initialize them as indicated by their data-adf- attributes.

Requirements:

  • .adf-region class
  • id attribute that is unique to the page. In most circumstances this would ideally be unique to all ADF regions in the application but that isn't always possible.
  • data-adf-region-type attribute with one of the below accepted region types (* indicates special region types and unlikely will be added to individual implementations):
    • form
    • grid
    • messages-window*
    • modules
    • overlay-grid*

Optional Attributes

  • data-adf-ajax-onshow is a simple boolean (literally true/false as the values) that will tell the framework to initiate an ajax request for this region when the page finishes loading and the framework itself has finished initializing itself.
  • data-adf-ajax-url is the default URL to use for ajax calls whether they are initialized onload (via the above attribute) or by some other event on the page.
Examples

Form Example

Form regions (ie elements with data-adf-region-type="form" in them must be a parent of the form element itself.

            <div class="adf-region"
                id="process-master-filters"
                data-adf-region-type="form"
                data-adf-ajax-onshow="true"
                data-adf-ajax-url="../json/process-master/filters.json">

                <form action="#process-master-grid" class="submit-on-change" method="post">

                </form>

            </div>
